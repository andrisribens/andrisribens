# andrisribens

Personal website with a weather forecast app. Stack: Next.js 16, React 19, TypeScript, pnpm.

## Key paths

| Path | Purpose |
|------|---------|
| `src/app/` | App Router pages and layouts |
| `src/app/weather/` | Weather route (server page, layout metadata) |
| `src/app/utilities/actions.ts` | Server actions — weather API, geocoding |
| `src/app/utilities/useDebounce.ts` | Client debounce hook |
| `src/components/` | UI components with co-located `.module.scss` |
| `src/styles/` | Global SCSS (variables, fonts) |
| `build-sass.mjs` | Compiles `src/components/**/*.scss` → `public/css/` |

## Commands

```bash
pnpm dev              # dev server (Next compiles SCSS automatically)
pnpm build            # build-sass + next build (prod)
pnpm run build-sass   # optional local mirror to public/css/
pnpm lint             # ESLint
```

## Conventions

- **Server components by default**; `'use client'` only when needed (hooks, localStorage, charts, router).
- **Server actions** in `actions.ts` for external API calls.
- **Weather flow:** `?place=` query param → `WeatherFactsServer` fetches place + weather → `WeatherFactsClient` renders charts.
- **Imports:** `@/` maps to `src/`.

## SCSS and Git workflow

### Source of truth

| What | Where |
|------|--------|
| Component styles | `src/components/<name>/<Name>.module.scss` |
| Global styles | `src/styles/` (`globals.scss`, `_variables.scss`, `_fonts.scss`) |
| Generated mirror | `public/css/` — **do not edit or commit** |

### How styles are compiled

1. **Next.js (primary)** — Components import `./Component.module.scss`. Next compiles SCSS at dev and build time. This is what the app uses.
2. **`build-sass.mjs` (mirror)** — `pnpm run build-sass` compiles the same SCSS files to `public/css/` with source maps. Nothing in the app imports these files; output is regenerated by `pnpm build` and stays out of git.

### When you change styles

1. Edit only `.scss` under `src/components/` or `src/styles/`.
2. Verify in `pnpm dev` — Next recompiles automatically; no extra step needed.
3. Commit **only** the `.scss` changes. Do not commit `public/css/`.

### Production

Deploy runs `pnpm build`, which compiles SCSS via Next.js (and regenerates the `public/css/` mirror locally). Committed CSS files are not required for prod.
